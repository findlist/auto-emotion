## 十、验收测试用例（核心场景）

### 场景 1：完整对战流程（自由乱斗）

```
1. 用户 A-H 分别注册登录
2. A 创建房间（自由乱斗），B-H 加入
3. 全员输入压力源（如"加班""堵车""KPI""房贷"等）
4. 房主开局，AI 生成怪兽 + 关卡 + 事件
5. 6-8 人乱斗 3-5 分钟
6. 实时分数同步
7. 随机事件触发
8. 最终排名下发
9. 战绩落库 + 收益结算
10. 8 人查询战绩
预期：全流程无报错，数据一致
```

### 场景 2：Boss 组队战

```
1. 用户 A-D 注册登录
2. A 创建房间（Boss 组队战），B-D 加入
3. 全员输入压力源
4. AI 生成情绪 Boss
5. 3-4 人组队攻击 Boss
6. 破坏场景物品积攒大招
7. Boss 击败全员获胜
8. 按贡献排名结算
预期：Boss 击败流程闭环，贡献排名正确
```

### 场景 3：手速竞速

```
1. 用户 A 单人进入手速竞速
2. 90 秒内完成微动作连击
3. 结算分数
4. 记入速度榜
预期：连击计分正确，榜单更新
```

### 场景 4：挂机 + 离线收益

```
1. 用户 A 进入挂机空间
2. 切换到"职场焦虑区"
3. 在线挂机 5 分钟，观察收益累计
4. 下线
5. 2 小时后上线
6. 领取离线收益
预期：离线收益 = 2 小时 × 挂机效率，数据正确
```

### 场景 5：AI 兜底

```
1. 正常进入房间
2. 模拟 AI API 超时（断网或 mock）
3. 房主开局
预期：10 秒后使用兜底怪兽/关卡/事件，对战正常进行
```

### 场景 6：断线重连

```
1. 对战中用户 A 断网
2. 3 秒内重连
预期：A 自动重连，补齐错过的最后 10 条事件，可继续对战
```

### 场景 7：并发结算

```
1. 同一对局触发 2 次结算请求
预期：只结算一次，第二次返回幂等结果
```

### 场景 8：越权访问

```
1. 用户 A 查询用户 B 的战绩
预期：返回 403
2. 用户 A 修改用户 B 的资料
预期：返回 403
```

### 场景 9：房间异常

```
1. 对战中房主退出
预期：房主转移，对战继续
2. 全员退出
预期：房间 5 分钟后自动销毁
```

### 场景 10：情绪自适应反馈

```
1. 玩家 A 在对战中高频点击（连击 20+）
2. 系统每 5 秒分析一次
预期：特效烈度提升到 4-5 档，反馈增强
3. 玩家 B 低频点击
预期：特效烈度维持 1-2 档
```

### 场景 11：商业化购买

```
1. 用户 A 购买皮肤（金币充足）
2. 同一 idempotencyKey 5 秒内重复请求
预期：只购买一次，第二次返回幂等结果
3. 用户 A 装备皮肤
4. 同类型再装备另一个
预期：旧装备自动取消
```

### 场景 12：通行证升级

```
1. 用户 A 完成对局获得通行证经验
2. 经验达标自动升级
3. 解锁对应奖励
预期：通行证等级 +1，奖励发放
```
