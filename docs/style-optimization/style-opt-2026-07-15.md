## 具体修改内容

### 1. 新增 CSS 工具类（index.css 第七轮扩展）

在 `index.css` 末尾新增"第七轮样式优化扩展"区块，包含 3 个关键帧与 7 类工具类，为本次各页面的氛围层次与微交互增强提供统一的样式基础。

#### 1.1 关键帧

| 关键帧 | 用途 | 设计原因 |
| --- | --- | --- |
| `scanlineMove` | 扫描线缓慢下移 | 复古街机 CRT 视觉语言，与 Neo-brutalism 硬边美学同源 |
| `metalShine` | 金属光泽斜向扫过 | 让 Top3 奖牌"活起来"，强化冠军视觉重量 |
| `premiumSweep` | 高级奖励流光 | 暗示"尊贵/稀有"，与 👑 高级标识配色一致 |

#### 1.2 工具类

| 类名 | 类型 | 用途 |
| --- | --- | --- |

### 2. 战斗页 · CRT 扫描线氛围层（battle.tsx）

**问题**：战斗画布为纯色背景（`backgroundColor: 0xfff8e7`），缺乏游戏沉浸感。HUD 信息条（`hud-chip`）已有 `backdrop-blur` 现代感，但画布本体与 Neo-brutalism 的"硬边+网格"美学缺乏呼应。

**修改**：在画布容器内新增 `<div className="scanlines-overlay" aria-hidden="true" />`，叠加极低透明度的水平扫描线 + 缓慢下移的高光条。

**设计原因**：
- 扫描线是复古街机游戏的经典视觉语言，与项目"情绪爆破局"的宣泄解压调性契合
- 透明度仅 0.04，不干扰 PixiJS 渲染的游戏内容可读性
- `pointer-events: none` 确保不阻挡画布的鼠标射击/右键技能交互
- `z-index: 5` 低于 HUD 覆盖层（z-10）与遮罩层（z-20/30/50），仅作为画布底色氛围

### 3. 首页 · 快捷入口卡片交互 + 导航激活点（home.tsx）

#### 3.1 快捷入口三联卡片加 card-hover

**问题**：首页"每日奖励/排行榜/武器库"三个快捷入口卡片为静态 `<div>`，无任何 hover 反馈。用户视觉上无法判断是否可交互。

**修改**：三个卡片均添加 `card-hover` 类，hover 时卡片浮起（`translate(-2px,-2px)`）+ 阴影扩大。

**设计原因**：即使暂未绑定 `onClick`，`card-hover` 传递"可交互"视觉信号，为后续绑定点击事件预留交互预期，与"更多功能"网格按钮的 hover 反馈一致。

#### 3.2 底部导航激活态小圆点

**问题**：底部导航激活态仅靠 `bg-pink` + `-translate-y-[1px]` 区分，小屏上 1px 位移难以感知。

**修改**：激活态按钮添加 `nav-active-dot` 类 + `relative` 定位，顶部显示 6px 黄色小圆点（带 ink 描边）。

**设计原因**：小圆点提供额外的激活态视觉锚点，黄色与 pink 激活背景形成 Neo-brutalism 标志性的撞色对比，在任意屏幕尺寸下都清晰可辨。

### 4. 挂机页 · 属性卡片左侧色条（idle.tsx）

**问题**：挂机页"升级角色属性"区域有 6 个属性卡片（生命/攻击/防御/暴击率/暴击伤害/挂机效率），视觉同质化严重，玩家难以快速定位目标属性。

**修改**：每个属性卡片根据 `field.key` 动态添加 `attr-bar-${field.key}` 类，左侧显示 6px 色条：

| 属性 | 色相 | 语义 |
| --- | --- | --- |
| 生命 hp | pink 粉 | 生命值=核心生存，与首页头像 pink 呼应 |
| 攻击 attack | orange 橙 | 攻击=进攻性，橙色暖色调 |
| 防御 defense | mint 薄荷 | 防御=守护，薄荷绿冷静感 |
| 暴击率 crit_rate | yellow 黄 | 暴击=高光时刻，黄色醒目 |
| 暴击伤害 crit_damage | #ff8c42 橙红 | 暴伤=更强攻击，橙红比 orange 更深一档 |
| 挂机效率 efficiency | ink 黑 | 效率=机制性属性，黑色中性 |

**设计原因**：色条是 Neo-brutalism 常见的分类视觉手段（粗边框+色块），6 种色相让卡片在滚动时形成色带节奏，玩家扫一眼即可定位目标属性，减少认知负担。

### 5. 排行榜 · Top3 奖牌金属光泽（leaderboard.tsx）

**问题**：Top3 奖牌已用 `medal-gold/silver/bronze` 渐变模拟金属质感，但渐变是静态的，缺乏"冠军"应有的视觉重量。Top1 虽有 `animate-badge-pulse` 脉冲，但脉冲是阴影扩散，不增强金属质感。

**修改**：Top1-3 奖牌均添加 `medal-shine` 类，叠加缓慢斜向扫过的白色高光（5s 周期，1s 延迟）。Top1 同时保留 `animate-badge-pulse`。

**设计原因**：
- 金属光泽扫过模拟光线在奖牌表面的反射，让静态渐变"活起来"
- Top1-3 均加光泽，但 Top1 额外的脉冲阴影进一步强化冠军独特性
- 5s 周期足够缓慢，不干扰排名浏览的专注度
- `overflow: hidden` 确保光泽不溢出圆形奖牌边界

### 6. 赛季通行证 · 高级奖励流光（season-pass.tsx）

**问题**：高级奖励栏与免费奖励栏仅靠 `bg-yellow/20` 底色区分，视觉差异不够强烈。高级通行证是付费内容，需更强的"尊贵/稀有"视觉暗示。

**修改**：未领取的高级奖励栏添加 `premium-shine` 类，叠加黄色流光（3.5s 周期，2s 延迟）。已领取态不加流光。

**设计原因**：
- 黄色流光与 👑 高级标识、`bg-yellow/20` 底色配色一致，强化"高级"视觉品牌
- 仅未领取态加流光，与上一轮建立的"claimed = 静态归档"理念一致——已领取不应再有动态预期
- 流光从左向右扫过，暗示"待获取"的方向性，引导用户点击领取

### 7. 登录/注册页 · 输入框焦点光晕（login.tsx / register.tsx）

**问题**：原输入框 focus 仅 `focus:ring-2 focus:ring-pink/30`，30% 透明度的 ring 在 cream 背景上对比度偏弱，焦点态不够醒目。

**修改**：
- 登录页 2 个输入框：替换为 `input-focus-pink`（cream 间隙 + pink 实色光晕 + ink 硬阴影）
- 注册页 4 个输入框：替换为 `input-focus-mint`（cream 间隙 + mint 实色光晕 + ink 硬阴影）

**设计原因**：
- 实色光晕（pink/mint）比 30% 透明度 ring 对比度更强，符合无障碍可感知性原则
- cream 间隙让光晕与 ink 边框分离，形成 Neo-brutalism 标志性的"分层描边"效果
- 登录用 pink、注册用 mint，与两页的主题色一致（登录标题 pink 徽章、注册标题 mint 徽章）

### 8. 大厅页 · 按钮 emoji 图标（lobby.tsx）

**问题**：大厅页 3 个主操作按钮（创建房间/快速匹配/加入房间）为纯文字，视觉同质化，用户需逐个阅读才能区分功能。

**修改**：
- 创建房间 → 🚪（门，暗示"新建空间"）
- 快速匹配 → ⚡（闪电，暗示"快速"）
- 加入房间 → 🔑（钥匙，暗示"凭房间号进入"）

**设计原因**：emoji 图标让按钮功能一目了然，减少文字阅读成本。三个图标语义与按钮文字互补，aria-hidden 屏蔽避免屏幕阅读器冗余朗读。

### 9. 房间页 · 模式选择 emoji 图标（room.tsx）

**问题**：房主设置区的 3 个游戏模式按钮（Boss 组队战/自由乱斗/手速竞速）为纯文字，模式差异缺乏视觉暗示。

**修改**：`GAME_MODES` 常量新增 `emoji` 字段，按钮内显示对应图标：
- Boss 组队战 → 🐉（龙，暗示 Boss 怪兽）
- 自由乱斗 → ⚔️（交叉剑，暗示对战）
- 手速竞速 → ⚡（闪电，暗示速度）

**设计原因**：与大厅页按钮图标风格统一，emoji 让模式选择更直观。🐉/⚔️/⚡ 与 battle.tsx 的 `MODE_LABEL` 语义对应，玩家从大厅到房间的模式选择体验连贯。
