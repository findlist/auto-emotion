## 具体修改内容

### 1. 登录/注册页表单卡片不可见边框修复

**文件**: `src/pages/login.tsx`、`src/pages/register.tsx`

**问题**: 表单卡片使用 `border-4 border-cream`，边框色与背景色（`bg-cream`）完全相同，导致 Neo-brutalism 硬边框隐形。项目其他所有卡片均使用 `border-ink` 形成可见的黑色硬边框，此处为遗漏。

**修改**: `border-cream` → `border-ink`

**优化前**:
```
bg-cream border-4 border-cream shadow-[8px_8px_0_#1a1a1a]
```

**优化后**:
```
bg-cream border-4 border-ink shadow-[8px_8px_0_#1a1a1a]
```

**效果**: 表单卡片现在有清晰的黑色硬边框，与项目其他卡片（大厅角色卡、房间玩家列表卡、商店商品卡等）视觉风格统一。

---

### 2. 大厅/房间页错误提示配色统一

**文件**: `src/pages/lobby.tsx`、`src/pages/room.tsx`

**问题**: 错误提示使用 `bg-red-100 border-red-500 text-red-600`，红色系完全脱离项目 Neo-brutalism 调色板（pink/yellow/orange/mint/ink/cream）。登录/注册页的错误提示已正确使用 `bg-pink/10 border-pink text-pink`，此处不一致。

**修改**: red 系 → pink 系，并补充 `shadow-[3px_3px_0_#1a1a1a]` 硬阴影

**优化前**:
```
bg-red-100 border-2 border-red-500 px-4 py-2 animate-shake
text-red-600 font-mono text-sm
```

**优化后**:
```
bg-pink/10 border-2 border-pink px-4 py-2 animate-shake shadow-[3px_3px_0_#1a1a1a]
text-pink font-mono text-sm
```

**效果**: 错误提示配色与设计系统一致，硬阴影增强 Neo-brutalism 层次感，与登录页错误提示风格统一。

---

### 3. 房间页取消准备按钮配色修复

**文件**: `src/pages/room.tsx`

**问题**: 取消准备按钮使用 `bg-red-500 hover:bg-red-600`，红色脱离调色板。准备按钮使用 `bg-mint`，取消应使用调色板内的警示色。

**修改**: `bg-red-500` → `bg-orange`（`#ff6b35`，调色板内暖色）

**优化前**:
```
bg-red-500 text-cream hover:bg-red-600
```

**优化后**:
```
bg-orange text-cream hover:bg-ink
```

**效果**: 取消操作使用 orange 暖色警示，与调色板一致，hover 变 ink 深色与其他按钮交互模式统一。

---

### 4. 首页底部导航激活态阴影修复

**文件**: `src/pages/home.tsx`

**问题**: 底部导航栏 `bg-ink`（深色 `#1a1a1a`），激活按钮使用 `shadow-[2px_2px_0_#1a1a1a]`（阴影色同为 `#1a1a1a`），深色阴影在深色背景上完全不可见，无法体现 Neo-brutalism 的浮起层次。

**修改**: 阴影色 `#1a1a1a` → `#ffd93d`（yellow 调色板色）

**优化前**:
```
bg-pink text-cream shadow-[2px_2px_0_#1a1a1a] -translate-y-[1px]
```

**优化后**:
```
bg-pink text-cream shadow-[2px_2px_0_#ffd93d] -translate-y-[1px]
```

**效果**: 激活的导航项底部出现黄色硬阴影，在深色导航栏上清晰可见，强化浮起感与视觉层次。

---

### 5. 挂机页属性数值框对比度改善

**文件**: `src/pages/idle.tsx`

**问题**: 战力展示区为 `bg-ink`（深色），属性框使用 `bg-ink/80`（80% 透明度的同色）+ `border-cream/10`（10% 透明度边框）。同色叠加在深色背景上几乎不可见，边框也过弱，四个属性框边界模糊。

**修改**: `bg-ink/80` → `bg-cream/10`，`border-cream/10` → `border-cream/20`

**优化前**:
```
bg-ink/80 px-2 py-1 rounded border border-cream/10
```

**优化后**:
```
bg-cream/10 px-2 py-1 rounded border border-cream/20
```

**效果**: 浅色调底色在深色背景上形成可见的层次区分，边框透明度提升至 20% 让属性框边界清晰可辨，数值信息更易读取。

## 优化前后对比说明

### 设计系统一致性

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 边框配色 | 登录/注册卡片边框隐形 | 所有卡片统一 `border-ink` 硬边框 |
| 错误提示 | 3 种风格（red/pink/混用） | 全部统一 pink 系 + 硬阴影 |
| 按钮配色 | red 系脱离调色板 | 全部使用调色板内 6 色 |
| 阴影可见性 | 深色背景上深色阴影隐形 | 深色背景用浅色阴影 |
| 层次对比 | 深色区属性框不可辨 | 浅色底形成清晰层次 |

### 视觉表现力提升

1. **Neo-brutalism 风格一致性**: 修复了 4 处脱离设计系统的配色/边框问题，全站视觉语言统一
2. **层次感增强**: 深色背景区域（导航栏、战力区）的元素现在有可见的阴影和边界，浮起效果生效
3. **信息可读性**: 挂机页属性数值框从几乎不可见变为清晰可辨，玩家读取战力数据更轻松
4. **错误反馈一致性**: 所有页面的错误提示现在使用统一的 pink 系配色 + 硬阴影 + shake 动画
