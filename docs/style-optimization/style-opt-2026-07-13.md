## 具体修改内容

### 1. 调色板一致性优化（核心）

项目设计系统基于 Neo-brutalism 风格，主调色板为：`pink / yellow / orange / ink / cream / mint`。审查发现多处页面使用了脱离调色板的 Tailwind 默认色（`green-500`、`red-500`、`gray-300`、`gray-400`、`amber-50`、`amber-600`、`amber-700`），破坏了视觉一致性。

#### 1.1 idle.tsx · 锁定态与停用按钮

**问题**：挂机区域按钮的锁定态用 `bg-gray-300 text-gray-500`，技能停用按钮用 `bg-red-500`，均脱离调色板。

**修改**：
- 锁定态：`bg-gray-300 text-gray-500` → `bg-ink/5 text-ink/40 border-2 border-dashed border-ink/30 cursor-not-allowed`
  - 用 ink 灰阶 + 虚线边框表达"未解锁"语义，符合 Neo-brutalism 的硬边框风格
- 技能停用按钮：`bg-red-500` → `bg-orange`
  - 复用调色板内的 orange（#ff6b35）表达"警示/停用"语义，与攻击/能量色一致

#### 1.2 tasks.tsx & achievements.tsx · 已领取状态

**问题**：已领取状态用 `bg-green-500` / `border-green-500` / `text-green-600`，与"可领取"的 `mint` 色混淆，且 green 脱离调色板。

**修改**：
- 边框：`border-green-500` → `border-ink/40`
- 进度条：`bg-green-500` → `bg-ink/40`
- 状态标签：`bg-green-500 text-cream` → `bg-ink text-cream/70`
- 已领取标签（achievements）：`text-green-600 bg-green-500/10` → `text-ink/60 bg-ink/10`

**设计原因**：已领取是"归档态"，用 ink 灰阶（深色低对比）表示已结束；可领取用 mint（饱满成功色）吸引点击。两者色彩完全不同，语义清晰，且都在调色板内。

#### 1.3 season-pass.tsx · 已领取奖励

**问题**：免费/高级奖励领取后用 `bg-green-500/20 border-green-500/40` 和 `text-green-600 bg-green-500/10`。

**修改**：
- 已领取背景：`bg-green-500/20 border-green-500/40` → `bg-mint/20 border-mint/40`
- 已领取标签：`text-green-600 bg-green-500/10` → `text-ink/60 bg-mint/20`

**设计原因**：赛季通行证的"已领取"是"收获"语义（玩家获得了奖励），用 mint（成功色）的淡化版表达，与未领取的 `bg-ink/5`（中性）和高级未领取的 `bg-yellow/20`（金币色）形成层次。

#### 1.4 leaderboard.tsx · Top2/3 行背景

**问题**：Top2 用 `bg-gray-100 border-gray-400`，Top3 用 `bg-amber-50 border-amber-600`，与 medal-silver/medal-bronze 奖牌体系不一致，且脱离调色板。

**修改**：
- Top2：`bg-gray-100 border-gray-400` → `bg-ink/10 border-ink/40`（银 → ink 灰阶）
- Top3：`bg-amber-50 border-amber-600` → `bg-orange/10 border-orange/60`（铜 → orange）

**设计原因**：与 medal-gold/silver/bronze 奖牌体系呼应——金(yellow)/银(ink 灰阶)/铜(orange)，前三名形成暖→冷→暖的色彩层次，且全部在调色板内。

#### 1.5 battle.tsx · 结算弹窗奖牌色

**问题**：结算弹窗前三名用 `text-gray-400` / `text-amber-700` 和 `border-gray-400 bg-gray-100` / `border-amber-700 bg-amber-50`，与排行榜 medal 体系不一致。遮罩用 `bg-black/60` 与断线重连遮罩的 `bg-ink/60` 不一致。

**修改**：
- 奖牌文字色：`text-gray-400` → `text-ink/50`，`text-amber-700` → `text-orange`
- 行背景：`border-gray-400 bg-gray-100` → `border-ink/40 bg-ink/5`，`border-amber-700 bg-amber-50` → `border-orange/60 bg-orange/10`
- 遮罩：`bg-black/60` → `bg-ink/60`（与断线重连遮罩统一）

### 2. 响应式布局优化

#### 2.1 shop.tsx · 商品卡片窄屏单列

**问题**：商品卡片固定 `grid-cols-2`，在 375px 窄屏上两个卡片挤在一起，emoji 和文字可读性差。

**修改**：`grid grid-cols-2 gap-3` → `grid grid-cols-1 sm:grid-cols-2 gap-3`
- 窄屏（<640px）单列展示，宽屏（≥640px）双列

#### 2.2 lobby.tsx · 标题与信息卡小屏适配

**问题**：
- 标题 `text-5xl`（48px）在 375px 屏上"游戏大厅"四字接近换行
- 角色信息卡 `w-80`（320px）+ 父容器 `p-8`（64px padding），在 375px 屏会溢出
- 操作按钮区 `w-64`（256px）同理

**修改**：
- 标题：`text-5xl` → `text-4xl sm:text-5xl`（小屏 36px，大屏 48px）
- 角色信息卡：`w-80` → `w-80 max-w-full`（保留目标宽度但不超过视口）
- 操作按钮区：`w-64` → `w-64 max-w-full`

### 3. 视觉表现力增强

#### 3.1 home.tsx · 雷达卡片标题装饰线

**问题**：首页雷达卡片标题"压力分布雷达"是纯文本，缺乏"雷达/仪表"的科技感。

**修改**：标题左右各加一条 `h-px w-8 bg-cream/30` 装饰线，用 flex 居中布局。

```jsx
<div className="flex items-center justify-center gap-2 mb-1">
  <span className="h-px w-8 bg-cream/30" aria-hidden="true" />
  <p className="font-cn text-xl">压力分布雷达</p>
  <span className="h-px w-8 bg-cream/30" aria-hidden="true" />
</div>
```

**设计原因**：装饰线呼应"雷达扫描"的视觉语言，增强深色卡片内的科技氛围；`bg-cream/30` 半透明与父级 `bg-ink text-cream` 协调，不抢夺雷达图本身的视觉焦点。

## 设计原则总结

本次优化遵循以下原则：

1. **调色板收敛**：所有颜色回归项目设计令牌（`pink/yellow/orange/ink/cream/mint`），消除 Tailwind 默认色对设计系统的干扰
2. **语义化色彩分层**：用同一色系的不同明度区分状态（如 mint 饱满=可领取，mint/20 淡化=已领取；ink 灰阶=归档态）
3. **响应式渐进增强**：移动优先，窄屏单列/小尺寸，宽屏双列/大尺寸，用 `sm:` 断点控制
4. **克制装饰**：视觉增强（如标题装饰线）仅用于强化内容语义（雷达科技感），不为装饰而装饰
5. **功能零侵入**：所有修改限于样式层，业务代码一字不动
