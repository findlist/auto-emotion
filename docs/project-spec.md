1. 项目概述
1.1 项目背景
当代18-35岁青年群体长期面临职场内卷、学业压力、生活焦虑、社交内耗等多重情绪压力，现存情绪解压渠道普遍存在门槛高、即时性差、趣味性弱、无社交陪伴感等痛点。
传统单机解压游戏缺乏长线留存与社交互动，竞技类游戏存在挫败感、无法舒缓情绪，心理咨询门槛过高、普及度低。基于该社会痛点，本项目依托AI能力打造情绪具象化解压网游，将现实压力转化为可击碎、可互动、可消解的游戏内容，为用户提供低门槛、即时性、社交化的情绪释放渠道。
1.2 项目核心定位
- 产品形态：纯网页端（H5）轻量级网游，无需下载、即开即玩
- 核心体验：双节奏玩法——后台挂机长线养成、上线短时多人乱斗解压
- 产品内核：情绪具象化 + 无压力竞技 + 轻量化养成 + 社交陪伴
- 目标用户：18-35岁都市职场人、学生群体，有碎片化解压、轻度社交需求的用户
- 使用场景：午休碎片时间、下班放松、朋友轻量开黑、通勤挂机养成
1.3 核心差异化卖点
1. AI情绪怪兽生成：将加班、KPI、堵车、催婚等真实压力源，AI动态生成专属情绪Boss，实现压力具象化消解
2. 零挫败竞技机制：无死亡惩罚、无段位碾压、无高强度竞技压力，主打破坏、互动、解压，适配所有玩家
3. 双节奏适配全用户：佛系用户可挂机养成，活跃用户可短时对战，兼顾长线留存与即时体验
4. 轻量化低门槛：H5免安装、操作简单、单局3-5分钟，适配碎片化场景

2. 整体系统架构
2.1 架构模式
采用Monorepo 前后端分离架构，前端轻量化游戏渲染，后端业务逻辑统一管控，WebSocket支撑实时对战，数据库持久化数据，Redis缓存临时状态，AI服务支撑动态内容生成。
核心架构理念：前端渲染、后端裁判、状态分层存储、AI动态内容，兼顾低延迟游戏体验与服务稳定性。
2.2 技术栈明细
前端技术栈
- 核心框架：React 19 + TypeScript + Vite
- 游戏渲染：PixiJS 8（高性能2D WebGL渲染）
- 样式方案：TailwindCSS 4
- 状态管理：Zustand 5
- 实时通讯：Socket.IO 客户端
后端技术栈
- 服务框架：Express 5 + TypeScript
- 实时服务：Socket.IO 4（房间管理、实时对战）
- 数据库：PostgreSQL 16（业务数据持久化）
- 缓存服务：Redis 7（房间状态、挂机缓存、会话管理）
- AI能力：大模型API（怪兽、关卡、随机事件动态生成）
2.3 架构分层说明
1. 前端表现层：负责页面UI、PixiJS游戏场景渲染、用户操作监听、本地状态同步
2. 实时通讯层：WebSocket实现房间联动、对战同步、实时事件广播
3. 后端业务层：用户、挂机、对战、商城、任务、社交核心业务逻辑处理
4. AI内容层：动态生成情绪怪兽、对战关卡、随机对局事件
5. 数据存储层：PostgreSQL持久化核心数据，Redis缓存热数据与临时状态

3. 核心业务模块设计
3.1 用户体系模块（P0核心）
支撑所有业务的基础模块，实现用户注册、登录、鉴权、个人资料管理，保障全场景用户身份统一。
- 账号体系：手机号注册、密码登录、JWT鉴权、Token无感刷新
- 账号安全：登录限流、密码哈希存储、登出黑名单拦截
- 资料管理：昵称、头像、个性签名、等级、战力、对战积分展示与修改
3.2 挂机养成模块（长线留存核心）
面向佛系用户的长线养成体系，离线亦可积累收益，支撑用户长期留存，搭配多维度成长体系。
- 挂机区域：五大情绪主题挂机场景，随等级解锁，难度与收益逐级提升
- 离线收益：支持12小时离线收益上限，上线一键领取，后端精准计算防篡改
- 成长体系：角色等级、战力、解压武器、主动/被动技能、宠物挂件多维养成
- 挂机日志：记录掉落、收益明细，可视化挂机成长进度
3.3 多人对战模块（核心体验P0）
项目核心解压玩法，支持房间创建、匹配、多人实时对战，提供短时高强度情绪释放体验。
- 房间系统：自定义房间、快速匹配、玩家准备、房主开局、房间状态管理
- 断线重连：WebSocket断线自动重连、房间状态恢复、对局进度保留（当前收尾优化项）
- 状态同步：玩家操作、分数、对局事件全房间实时广播
3.4 AI动态内容模块（项目特色P0）
区别于传统游戏的核心特色，基于用户压力源动态生成对局内容，实现千人千面的解压体验。
- 情绪怪兽生成：根据用户输入的压力关键词，AI生成专属Boss属性、技能、弱点、外观
- 动态关卡生成：随机场景布局、可破坏物、障碍物、刷新点位，每局体验不重复
- 随机事件生成：对局中触发各类趣味情绪事件，增强解压趣味性与随机性
- 情绪自适应：根据用户操作节奏动态调整对局强度，适配不同解压需求
3.5 结算战绩模块
完成对局数据沉淀、收益结算、战绩归档，支撑成长体系与排行榜数据。
- 实时结算：对局结束自动计算积分、排名、经验、金币、道具掉落
- 战绩归档：单局详情、对局模式、参与玩家、对局时长、MVP记录持久化存储
- 数据统计：支撑各类排行榜、个人对战数据统计
3.6 社交与排行模块（P1）
- 好友体系：好友申请、添加、私聊、对局组队邀请
- 多维度排行榜：战力榜、对战积分榜、手速竞速榜、好友榜单
3.7 任务与成就模块（P1）
- 每日任务：对战、挂机、社交三类日常任务，完成可领取奖励
- 成就体系：覆盖养成、对战、社交全场景成就，解锁专属荣誉
- 赛季通行证：阶段性赛季奖励体系，提升用户长线活跃
3.8 商城商业化模块（P1）
纯外观、功能性道具付费，无数值碾压，保证游戏公平性。
- 售卖内容：角色皮肤、武器外观、挂机加速卡、装饰道具
- 权限体系：普通免费奖励 + 高级通行证付费奖励
- 物品管理：个人背包、装备穿戴、已购道具记录

4. 三大对战模式规格
4.1 情绪Boss组队战（PVE 3-4人）
- 核心玩法：多人组队共同击碎AI生成的情绪压力Boss
- 解压亮点：集体爆破具象化压力源，全屏特效释放情绪
- 对局时长：3-5分钟
- 结算规则：全员获胜，根据伤害贡献、破坏积分排名发放奖励
4.2 解压自由乱斗（PVP 6-8人）
- 核心玩法：限时内自由破坏场景、互动对战，累计积分
- 核心规则：无死亡惩罚、无淘汰机制，仅击飞、粘滞趣味效果
- 解压亮点：肆意破坏、趣味互坑，零压力竞技体验
- 对局时长：3-5分钟
- 结算规则：按最终积分排名发放奖励
4.3 手速竞速挑战（单人/多人排行）
- 核心玩法：90秒限时捏泡泡、砸西瓜、撕胶带等高频微操作挑战
- 解压亮点：高频连击、即时反馈，极致手感解压
- 对局时长：90秒
- 结算规则：按连击倍率、总得分排名，计入全局速度榜单

6. 统一技术规范
6.1 API 统一响应格式
所有后端接口统一返回结构，保证前端解析逻辑统一、异常处理标准化。
interface ApiResponse<T = any> {
  code: number;        // 200成功，其余为异常
  message: string;     // 提示文案
  data: T;             // 业务数据
}

// 分页专用结构
interface PageResponse<T> {
  list: T[];
  total: number;
  page: number;
  pageSize: number;
  totalPages: number;
  hasNext: boolean;
}
6.2 通用错误码规范
- 200：成功
- 201：创建成功
- 400：请求参数错误
- 401：未授权/Token失效
- 403：权限不足
- 404：资源不存在
- 409：状态冲突/重复操作
- 422：参数校验失败
- 429：请求频繁限流
- 500：服务器内部异常
6.3 命名规范
- 文件/接口路径/WebSocket事件：kebab-case
- 变量/函数：camelCase
- 类/接口：PascalCase
- 常量：UPPER_SNAKE_CASE
- 数据库表/字段：snake_case
6.4 前端体验强制规范
所有页面、组件必须标配基础体验能力，保证整体体验统一精致：
- 响应式适配：基于 max-w-2xl mx-auto 实现移动端自适应（当前收尾优化项）
- 状态兜底：加载动画、空状态提示、错误Toast提示全覆盖
- 交互防护：关键操作确认弹窗、表单校验、重复提交拦截（当前收尾优化项）

7. 质量与安全规范
7.1 数据安全规则
- 资金、经验、道具变更接口强制数据库事务+行锁，保证数据一致性
- 付费、领奖接口实现5秒幂等去重，防止重复提交
- 用户密码bcrypt加盐哈希存储，禁止明文、弱加密
7.2 权限控制规则
- 全局JWT鉴权，所有业务接口强制登录校验
- 水平权限隔离：用户仅可操作自身账号、数据、道具
- 登出、封禁用户即时失效Token，Redis黑名单拦截
7.3 异常兜底机制
- 房间空置5分钟自动解散，对局超时自动结算，避免脏数据
- WebSocket断线自动重连，补齐离线事件，恢复对局状态
- 第三方依赖异常自动降级，不阻塞核心业务运行

10. 部署规范

基于Docker Compose一键部署，环境隔离、快速启动、适配本地与服务器环境。

docker compose up -d
