# 情绪爆破局 - 功能规格说明（spec.md）

> 本文档定义功能规格的 Why / What Changes / Requirements，作为开发依据。

---

## 一、Why（为什么做）

### 1.1 痛点
- 都市人群压力大，缺乏即时、社交化、低门槛的情绪释放渠道
- 单机解压游戏无社交快感、无长线目标，现有派对游戏不解压、有挫败感
- 心理咨询门槛高，无法覆盖"亚健康"人群的日常解压需求
- 现有挂机游戏重数值碾压、付费深，无解压情绪价值

### 1.2 价值
- **社会价值**：零成本、即时可用、长线陪伴的情绪释放出口，缓解都市心理亚健康（社会公益加分）
- **商业价值**：双节奏体验兼顾挂机党与对战党，留存高；AI 生成内容降低运营成本；外观商业化空间大

### 1.3 差异化
- **AI 情绪怪兽生成**：把玩家压力源变成专属 Boss，形成情感投射
- **双节奏体验**：挂机养成提供长线目标，多人乱斗提供即时解压，兼顾两类玩家
- **零挫败竞技**：无死亡惩罚、无段位压力，主打破坏得分与趣味互动

## 二、What Changes（要做什么）

### 2.1 核心功能变更

#### 变更 1：用户体系
- 新增手机号注册/登录、JWT 鉴权、个人资料管理
- 新增 JWT 黑名单登出机制

#### 变更 2：挂机养成系统（长线留存核心）
- 新增挂机空间：5 个情绪区域（职场/生活/学业/社交/存在焦虑），按等级解锁
- 新增离线收益：上限 12 小时，上线一键领取
- 新增 4 大成长线：角色等级、解压武器、技能词条、宠物挂件
- 新增挂机日志：近期掉落记录

#### 变更 3：多人对战系统
- 新增房间创建/加入/快速匹配
- 新增房间状态机：等待 → 准备 → AI 生成 → 对战中 → 结算 → 关闭
- 新增 3 种对战模式选择：Boss 组队战 / 自由乱斗 / 手速竞速
- 新增 WebSocket 实时房间状态同步

#### 变更 4：AI 能力系统（评分关键）
- 新增压力源文本输入接口
- 新增情绪怪兽生成：关键词 → 怪兽形象/技能/弱点
- 新增动态关卡生成：场景布局/可破坏物分布/刷新点位
- 新增随机事件生成：对局中趣味干扰事件（老板弹窗、广告弹窗等）
- 新增情绪自适应反馈：根据玩家操作节奏动态调整特效烈度
- 新增关卡下发机制（WebSocket 广播）

#### 变更 5：对战核心
- 新增 Boss 组队战模式（PVE，3-4 人）
- 新增自由乱斗模式（PVP，6-8 人，无死亡惩罚）
- 新选手速竞速模式（单人/多人，90 秒微动作连击）
- 新增实时操作同步、分数同步、事件触发、最终排名

#### 变更 6：结算系统
- 新增对局记录落库
- 新增收益结算：经验/金币/掉落
- 新增战绩查询

#### 变更 7：社交系统（P1）
- 新增好友申请/列表
- 新增组队邀请
- 新增私聊

#### 变更 8：排行榜（P1）
- 新增战力榜/对战榜/好友榜/速度榜

#### 变更 9：任务成就系统（P1）
- 新增每日任务（对战/挂机/社交三类）
- 新增成就体系
- 新增赛季通行证（免费 + 高级）

#### 变更 10：商业化系统（P1，无数值付费）
- 新增商品列表（角色皮肤/武器皮肤/装饰/挂机加速卡）
- 新增购买、装备、我的物品
- 新增通行证购买

### 2.2 非功能变更

- 新增 Docker 一键部署
- 新增健康检查接口 `/health`
- 新增结构化日志
- 新增限流中间件
- 新增定时任务（离线收益计算、房间清理、赛季刷新）

## 三、Requirements（功能需求）

### 3.1 用户体系

#### REQ-U01 注册
- 输入：手机号（11 位）、密码（8-32 位，含字母+数字）、昵称（1-32 位）
- 校验：手机号格式正则、手机号唯一、密码强度
- 安全：密码 bcrypt 加盐哈希、注册接口限流（5 次/分钟）
- 响应：返回 JWT token + 用户信息

#### REQ-U02 登录
- 输入：手机号、密码
- 校验：手机号存在、密码匹配、账号未封禁
- 安全：登录接口限流（5 次/分钟）、连续失败 5 次锁定 15 分钟
- 响应：返回 JWT token（2 小时有效）+ refreshToken（7 天）
- 副作用：更新 `last_login_at`（用于离线收益计算）

#### REQ-U03 鉴权
- 所有业务接口必须携带 `Authorization: Bearer <token>`
- 全局认证中间件校验 JWT
- JWT 黑名单（Redis）校验登出 token
- 水平权限：用户只能操作自己的资源

#### REQ-U04 个人资料
- 查询：`GET /api/users/profile`
- 修改：`PUT /api/users/profile`（昵称、头像、签名）

### 3.2 挂机养成系统

#### REQ-I01 挂机状态查询
- 接口：`GET /api/idle/status`
- 返回：当前区域、挂机效率、累计时长、当前可领取收益预估
- 逻辑：实时计算从 `idle_started_at` 到现在的收益

#### REQ-I02 切换挂机区域
- 接口：`POST /api/idle/switch-area` `{ areaCode }`
- 校验：区域存在、用户等级 ≥ 解锁等级
- 逻辑：先结算当前区域收益，再切换到新区域，重置 `idle_started_at`

#### REQ-I03 离线收益领取
- 接口：`POST /api/idle/claim`
- 逻辑：计算从 `last_login_at` 到现在的离线收益，上限 12 小时
- 收益：经验、金币、装备碎片、技能道具（按区域掉落表）
- 事务：收益落库使用事务，更新 `last_login_at`
- 幂等：5 秒窗口内重复领取返回上次结果

#### REQ-I04 角色成长
- 接口：`GET /api/character/profile`
- 返回：等级、经验、战力、属性明细
- 升级：经验达到阈值自动升级，提升基础属性

#### REQ-I05 武器系统
- 列表：`GET /api/weapons`（用户拥有的武器）
- 升级：`POST /api/weapons/:id/upgrade`（消耗金币 + 碎片）
- 装备：`POST /api/weapons/:id/equip`（同类型只能装备 1 个）

#### REQ-I06 技能系统
- 列表：`GET /api/skills`（已解锁技能）
- 解锁：`POST /api/skills/:id/unlock`（满足解锁条件）
- 升级：`POST /api/skills/:id/upgrade`

#### REQ-I07 宠物挂件
- 列表：`GET /api/pets`
- 装备：`POST /api/pets/:id/equip`

#### REQ-I08 挂机日志
- 接口：`GET /api/idle/logs`
- 返回：近期 50 条掉落记录

### 3.3 多人对战系统

#### REQ-R01 创建房间
- 输入：房间名（可选）、最大人数（4-8）、模式、是否公开
- 逻辑：生成 6 位房间号，创建 Redis 房间状态
- 响应：房间号 + 房间信息

#### REQ-R02 加入房间
- 输入：房间号
- 校验：房间存在、未满员、状态为"等待"
- 逻辑：加入房间，广播 `room:state`

#### REQ-R03 快速匹配
- 输入：模式
- 逻辑：从匹配队列找未满房间，找不到则创建
- 超时：30 秒未匹配成功返回失败

#### REQ-R04 准备/取消
- 事件：`room:ready` `{ ready: boolean }`
- 开局条件：房主 + 全员准备 + 人数达标（Boss 战 ≥ 3，乱斗 ≥ 4，竞速 ≥ 1）

#### REQ-R05 退出房间
- 事件：`room:leave`
- 逻辑：移除玩家，若房主退出则转移房主，若空房则销毁

#### REQ-R06 模式选择
- 事件：`room:set-mode` `{ mode }`
- 仅房主可设置，广播 `room:state`

### 3.4 AI 能力系统

#### REQ-A01 压力源输入
- 事件：`room:submit-stress` `{ stressSource: string }`
- 校验：文本长度 1-100 字
- 逻辑：收集到房间内所有玩家提交后触发 AI 生成

#### REQ-A02 情绪怪兽生成
- 输入：房间内所有玩家压力源文本 + 难度（按平均战力）
- 调用：大模型 API，prompt 见 `server/src/ai/prompts/monster-generation.md`
- 输出：`MonsterConfig`（名/头像/血量/技能/弱点/压力标签/外观）
- 超时：10 秒未返回则使用兜底怪兽
- 兜底：预设 5 套通用怪兽配置按难度选取

#### REQ-A03 动态关卡生成
- 输入：怪兽配置 + 模式
- 输出：`LevelLayout`（种子/场景/可破坏物/刷新点/障碍物）
- 超时：10 秒未返回则使用兜底布局
- 兜底：预设 3 套布局按模式选取

#### REQ-A04 随机事件生成
- 输入：模式 + 对局时长
- 输出：`GameEvent[]`（事件列表，含触发时间）
- 数量：每局 2-4 个事件
- 兜底：预设 10 个通用事件随机选取

#### REQ-A05 情绪自适应反馈
- 输入：玩家实时上报的操作节奏（`game:rhythm-report`）
- 逻辑：分析点击频率、连击数，动态调整特效烈度（1-5 档）
- 输出：`game:effect-intensity` `{ intensity }`
- 频率：每 5 秒分析一次

#### REQ-A06 关卡下发
- 事件：`game:level-ready` `{ seed, monster, layout, events }`
- 广播给房间内所有玩家

### 3.5 对战核心

#### REQ-G01 Boss 组队战（PVE）
- 人数：3-4 人
- 时长：3-5 分钟
- 玩法：共同攻击 Boss，破坏场景物品积攒大招能量
- 操作：左键攻击/破坏、右键大招、滚轮切换道具，角色自动辅助走位
- 得分：对 Boss 伤害 + 破坏物品分
- 结算：Boss 击败全员获胜，按贡献排名

#### REQ-G02 自由乱斗（PVP）
- 人数：6-8 人
- 时长：3-5 分钟
- 玩法：攻击玩家 + 破坏场景物品累计积分
- 机制：被击中仅击飞/粘滞，无死亡
- 得分：攻击命中分 + 破坏物品分 + 道具整活分
- 结算：按积分排名

#### REQ-G03 手速竞速（单人/多人）
- 时长：90 秒
- 玩法：捏泡泡、撕胶带、砸西瓜等微动作连击
- 得分：连击数 × 倍率
- 结算：单人记入速度榜，多人按分数排名

#### REQ-G04 实时同步
- 操作：`game:action` 广播给同房间玩家
- 分数：每 2 秒上报一次，广播 `game:score-update`
- 事件：按 `triggerTime` 触发，广播 `game:event`

#### REQ-G05 对战结束
- 事件：`game:finish` `{ ranking, mvpId, rewards }`
- MVP：分数最高者（Boss 战为伤害最高）
- 落库：写 `game_records` + `game_record_players`

### 3.6 结算系统

#### REQ-S01 收益规则
- 经验：基础 50 + 排名加成（第 1 +100、第 2 +50、第 3 +30）+ MVP +50
- 金币：基础 20 + 排名加成
- 掉落：按区域掉落表随机
- 对战积分：第 1 +30、第 2 +15、第 3 +5、其他 -5（不低于 0）

#### REQ-S02 战绩查询
- 列表：`GET /api/game-records?page=1&pageSize=20`
- 详情：`GET /api/game-records/:id`
- 权限：只能查自己的战绩

### 3.7 社交系统（P1）

#### REQ-SO01 好友
- 申请：`POST /api/friends/apply` `{ userId }`
- 接受/拒绝：`POST /api/friends/:id/respond` `{ accept }`
- 列表：`GET /api/friends`
- 删除：`DELETE /api/friends/:id`

#### REQ-SO02 组队
- 邀请：`WS team:invite` `{ userId }`
- 接受：`WS team:accept` `{ fromUserId }`
- 逻辑：组队后可一键加入同一房间

#### REQ-SO03 私聊
- 发送：`WS chat:private` `{ toUserId, message }`
- 接收：`WS chat:private` `{ fromUser, message }`
- 校验：必须是好友

### 3.8 排行榜（P1）

#### REQ-L01 战力榜
- 数据：用户战力
- 缓存：Redis ZSET，10 分钟刷新
- 接口：`GET /api/leaderboard/power?limit=100`

#### REQ-L02 对战榜
- 数据：对战积分
- 缓存：Redis ZSET，10 分钟刷新

#### REQ-L03 好友榜
- 数据：好友对战积分
- 逻辑：查好友列表 + 积分排序

#### REQ-L04 速度榜
- 数据：手速竞速最高分
- 缓存：Redis ZSET，1 小时刷新

### 3.9 任务成就系统（P1）

#### REQ-T01 每日任务
- 刷新：每日 0 点定时任务刷新
- 数量：每日 3 个（对战/挂机/社交各 1）
- 查询：`GET /api/tasks/daily`
- 领奖：`POST /api/tasks/:id/claim`（进度达标）
- 进度更新：行为触发自动累加

#### REQ-T02 成就
- 查询：`GET /api/achievements`
- 类型：累计对战数、累计解压时长、击杀 Boss 数等
- 完成：自动检测，发放奖励

#### REQ-T03 赛季通行证
- 查询：`GET /api/season-pass`
- 经验：对战/任务获得通行证经验
- 升级：经验达标自动升级，解锁对应奖励
- 高级：`POST /api/season-pass/buy`（钻石购买）

### 3.10 商业化系统（P1，无数值付费）

#### REQ-SH01 商品列表
- 接口：`GET /api/shop/items?type=&rarity=`
- 分页：20/页

#### REQ-SH02 购买
- 接口：`POST /api/shop/buy` `{ itemId, idempotencyKey }`
- 校验：商品上架、货币充足、未拥有
- 事务：扣货币 + 写 user_items，使用 `FOR UPDATE` 行锁
- 幂等：5 秒窗口内相同 idempotencyKey 去重

#### REQ-SH03 装备
- 接口：`POST /api/shop/equip` `{ itemId }`
- 校验：已拥有
- 逻辑：同类型只能装备 1 个

#### REQ-SH04 通行证购买
- 接口：`POST /api/season-pass/buy`
- 校验：钻石充足、未购买过本赛季高级
- 事务：扣钻石 + 更新 `is_premium`

## 四、非功能需求

### 4.1 性能
- 单房间对战延迟 < 100ms（WebSocket）
- AI 生成 < 10 秒（含兜底）
- API 响应 P95 < 500ms
- 支持同时 100 个房间并发
- 离线收益计算 < 1 秒

### 4.2 可用性
- WebSocket 断线 3 秒内自动重连
- 重连后补齐最后 10 条事件
- 房间空置 5 分钟自动解散
- 对局超时 10 分钟自动结算
- 离线收益上限 12 小时

### 4.3 安全
- 所有密码 bcrypt 哈希
- 所有 SQL 参数化查询
- JWT 黑名单机制
- 敏感接口限流
- 环境变量启动校验
- 付费接口二次校验

### 4.4 可维护性
- 前后端 TypeScript 严格模式
- 后端分层：路由 → 服务 → 数据
- 结构化日志（JSON）
- `/health` 健康检查

## 五、验收标准（概要）

详见 `checklist.md`，核心验收点：

- [ ] 用户可完成注册→登录→进入大厅全流程
- [ ] 挂机系统可自动累计收益，离线收益可领取
- [ ] 武器/技能/宠物可升级装备，影响战力
- [ ] 3 种对战模式均可正常游玩
- [ ] AI 能根据压力源生成差异化怪兽和关卡
- [ ] 对局中随机事件可触发
- [ ] 情绪自适应反馈可动态调整特效
- [ ] 对局记录正确落库可查
- [ ] 收益结算事务一致
- [ ] Docker 一键部署可运行
