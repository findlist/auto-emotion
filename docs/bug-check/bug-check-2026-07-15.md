## 检查结论

本次检查共发现 **4 个 P0 + 18 个 P1** 问题并全部修复通过验证，另记录 3 个 P1 需评估问题。

### 修复亮点

1. **P0 schema 字段不匹配修复**（achievement-service / friend-service）：通过交叉验证数据库 schema（001_init.sql）发现代码与表结构字段类型不一致，这是最隐蔽的问题——测试 mock 了错误的字段名所以测试通过，但实际运行会报 SQL 字段不存在错误。

2. **P0 并发安全修复**（achievement-service / pet-service）：统一采用 `pg_advisory_xact_lock` + 事务内 recheck 模式防止重复领奖，采用原子 `UPDATE...WHERE gold >= $1 RETURNING` 防止重复扣款。

3. **P1 资源泄漏修复**（客户端游戏引擎）：三个游戏引擎（boss/brawl/speed）统一在 cleanup 中复位 screenShake，brawl 的 removePlayer 补齐 destroy，boss/brawl 的游戏结束补齐 isRunning=false，消除连续开局时的内存泄漏与状态残留。

4. **P1 异常处理链路完善**：全后端 11 个 service 文件的 ROLLBACK 统一加 try/catch 保护，rate-limit Redis 故障 fail-open，pets /buy 幂等控制。
