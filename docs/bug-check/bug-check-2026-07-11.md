## 记录的轻微问题（P2，未修复）

### 前端

- **boss-game.ts**：投射物销毁存在双重 destroy 风险（update 中 destroy 后 onDestructibleDestroyed 可能再次访问）
- **battle-scene.ts**：异步 init 未 await，`syncPlayers` 在 init 完成前调用可能使用未初始化的游戏实例
- **speed-game.ts**：`switchMiniGame` 未清理 pendingTimers，快速切换小游戏可能导致定时器泄漏
- **websocket/index.ts**：`waitForConnection` 的 reject 在 socket 未连接时可能过早触发
- **App.tsx**：`handleLogout` 未捕获错误，logout API 失败时用户卡在当前状态
- **brawl-game.ts**：玩家碰撞检测可能重复处理同一帧的多次碰撞
- **多个页面**：9 处 `react-hooks/set-state-in-effect` 警告（useEffect 中同步 setState）

### 后端

- **room-manager.ts**：`generateRoomId` 使用 `Math.random()`，碰撞概率虽低但非密码学安全
- **event-generator.ts**：洗牌算法分布有偏（Fisher-Yates 实现不完整）
- **level-generator.ts**：对 AI 返回数据校验过弱，未验证字段类型与范围
- **config/index.ts**：部分配置项默认值与环境变量不一致
- **settle-service.ts**：`ROLLBACK` 未防错，rollback 本身抛错会掩盖原始错误
- **match-service.ts**：timer 的 async 回调未 catch，异常会导致定时器静默失败
- **match-service.ts**：`getMatchStatus` 返回的等待人数多算 1
- **app.ts**：优雅关闭未 await 数据库连接池关闭
- **websocket/index.ts**：CORS 配置全开（origin: true），生产环境应收紧
